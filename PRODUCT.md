# Product

## Register

product

## Users

普通用户，涵盖个人和团队场景。使用时可能处于工作中（深度对话、团队协作）或日常快速问答。他们需要一个不干扰思考的对话空间——打开即用，界面不制造认知负担。核心任务：与 AI 进行流畅对话、管理多个会话、在团队中共享上下文、快速获取答案。

## Product Purpose

Pi Chat 是一个基于 PI Agent SDK 的多用户 AI 聊天应用，提供流式对话、会话管理、联网搜索和可配置 Agent 能力。成功标准：用户能无障碍地开始对话、高效管理会话、在需要时切换到团队协作模式，界面始终不干扰对话本身。

## Brand Personality

高效、专业、简洁。

不卖弄设计，不制造视觉噪音。界面像一把好用的工具——用户不会注意到它，只会注意到对话很顺畅。克制但不冷漠，专业但不死板。

## Anti-references

- 花哨的视觉效果、过多的装饰元素、渐变滥用
- 聊天室式的混乱界面：密集的消息气泡、五颜六色的头像、不停闪烁的在线状态
- 企业 IM 的沉重感：过多侧边栏、层级嵌套、功能入口堆砌
- 任何让用户觉得"设计感压过了内容"的界面

## Design Principles

1. **内容即界面。** 对话是核心内容，所有 UI 元素都应为阅读和输入服务，不与对话争夺注意力。装饰性元素能省则省。
2. **安静的专业感。** 不需要通过视觉花哨来证明"这是个好产品"。克制的排版、精确的间距、一致的节奏传递出的专业感，比任何装饰都有效。
3. **零学习成本。** 用户打开应用就知道该做什么。不需要引导、不需要解释按钮的含义。如果某个交互需要说明，重新设计这个交互。
4. **密度服务于效率。** 信息密度要够——用户不应该需要反复滚动或切换页面才能看到需要的内容。但密度来自精确的空间利用，不是来自把所有东西塞进一个屏幕。

## Accessibility & Inclusion

按 WCAG AA 基本标准执行。无特殊无障碍需求。确保基本对比度和键盘可达性即可，不作为核心设计约束。
